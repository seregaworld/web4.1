Здравствуйте, сделаны задания:
1. Реализовать Rest методы, используя Express.js (put, post, get, delete) для набора повторяющихся данных из макета (например для ленты новостей). Данные хранить в любой бд (подойдут mongodb, mysql или любые другие).
Если повторяющихся данных в макете нет, то можно добавить новый блок. Его оформление не важно, можно скопировать из другого блока макета.

В случае сомнений в том, есть ли подходящие данные - присылаете мне номер макета и задаете вопрос о том, для каких данных выполнить задание.

2. Реализовать Angular сервис для отправки запросов к серверу из прошлого задания.

3. Создать директиву, которая, используя сервис из прошлого задания, будет выводить данные.

4. Используя роутинг реализовать переход к странице с более детальной информацией о полученных данных. Например, страница новости. Для каждой новости генерировать свою страницу.
Важно сделать это универсально (не создавать вручную по странице для каждой новости, а формировать страницы на основе полученных с сервера данных). Для удобства можно в Express реализовать метод получения данных по id.

5. Создать страницу, на которой можно добавить новые данные. (использовать сервис из второго задания и метод post).

6. Создать страницу, на которой можно редактировать или удалить данные (использовать сервис из второго задания и методы put, delete).

Переход между страницами осуществляется кнопками меню(в левом вверхнем углу страницы): Home, Gallery, Categories,	Pages.

 Home отвечает за переход страницу макета home.html , там ничего для заданий нет.

Gallery является переход к главной страницы для заданий newsList.html, на ней выводятся все продукты коллекции(краткая информация), здесь можно по коду продукта вывести полную информацию о продукте.
КОД ПРОДУКТА: ИМЕЕТ ВВИД ТИПА 5919f2f138d39b1f0b5be446, это и есть id продукта.

Categorie является переход к страницы postList.html в которой можно добавить новый продукт.
Pages является переход к страницы putDelete.html в которой можно изменить/удалить продукт по коду продукта(пример:5919f2f138d39b1f0b5be446).
